# Criterios de Aceptación
 | ID y nombre de la historia del usuario  |AC-ID| Descripción del criterio de aceptación | Estado (completado,pendiente,en progreso)| 
| ------------- | ------------- | ------------- | ---|
|HU-1 Creación de Cuenta de Usuario|AC-1.1 Creación exitosa de una cuenta de usuario válida|``Dado`` un nombre de usuario válido inexistente `Cuando` creo una cuenta de usuario con este nombre de usuario `Y` una contraseña válida que coincida con una contraseña confirmada `Entonces` el sistema debería crear una cuenta nueva y mostrar un mensaje "Registrado con éxito"|Completado| 
| |AC-1.2 Creación de una cuenta fallida con un nombre de usuario existente|`Dado` un nombre de usuario ya existente `Cuando` creo una cuenta con este nombre de usuario `Y` el campo de contraseña `Y` confirmación de contraseña son válidas `Entonces` el sistema no debería permitir crear una cuenta `Y` debería mostrar un mensaje de error|Completado| 
| |AC-1.3 Creación de una cuenta fallida con una contraseña no válida|`Dado` un nombre de usuario válido inexistente `Cuando` un usuario crea una cuenta con este nombre de usuario `Y` una contraseña no válida `Y` la misma contraseña confirmada `Entonces` el sistema no debería permitir crear la cuenta `Y` debería mostrar un mensaje de error|Completado|
||AC-1.4 Prevención de creación de cuenta con un nombre de usuario malicioso|`Dado` un nombre de usuario no válido malicioso `Y` una contraseña válida `Cuando` un usuario crea una cuenta con este nombre de usuario `Entonces` el sistema no debería permitir crear la cuenta `Y` debería mostrar un mensaje de error |En progreso|
|HU-2 Visualizar Tablero|AC-2.1 Arranque exitoso de la interfaz gráfica|`Dado` que el usuario haya iniciado sesión `Cuando` el usuario cree una partida `Entonces` el sistema debe generar un tablero|Completado|
|HU-3 Iniciar sesión|AC-3.1 Inicio de sesión exitoso con credenciales válidas|`Dado` un nombre de usuario existente y contraseña válida `Cuando` inicio sesión con este nombre de usuario `Y` una contraseña válida que corresponda al nombre de usuario `Entonces` el sistema debería realizar el inicio de sesión `Y` cargar la pantalla principal del juego|Completado|
| |AC-3.2 Inicio de sesión fallido con un nombre de usuario que no existe|`Dado` un nombre de usuario no existente y una contraseña `Cuando` inicio sesión con este nombre de usuario y una contraseña `Entonces` el sistema no debería iniciar sesión `Y` debería mostrar un mensaje de error|Completado|
| |AC-3.3 Inicio de sesión fallido con una contraseña incorrecta|`Dado` un nombre de usuario existente y una contraseña no válida `Cuando` inicio sesión con este nombre de usuario y contraseña `Entonces` el sistema no debería iniciar sesión `Y` debería mostrar un mensaje  de error|Completado|
| |AC-3.4 Prevención de inicios de sesión con un nombre de usuario malicioso|`Dado` un nombre de usuario no válido malicioso y una contraseña válida `Cuando` yo(el adversario) inicio sesión con el nombre de usuario y la contraseña `Entonces` los intentos de inicio de sesión deberían fallar|Completado|
|HU-4 Cerrar sesión|AC-4.1 Cerrar sesión exitoso|`Dada` una sesión iniciada y ninguna partida en curso, `Cuando` doy click en "Cerrar sesión" `Entonces` el sistema terminará la sesión `Y` mostrará la pantalla de inicio (Inicio de sesión).|Completado| 
| |AC-4.2 Cerrar sesión fallido|`Dada` una sesión iniciada `Y` una partida en curso, `Cuando` doy click en "Cerrar sesión" `Entonces` el sistema mostrará un mensaje diciendo "Partida en curso.¿Estás seguro que deseas finalizar la sesión?"|En progreso|
|HU-5 Crear una Partida|AC-5.1 Crear una partida contra otro usuario exitoso|`Dado` un usuario que ha iniciado sesión `Cuando` doy click en "Crear Partida" -> "Contra otra persona" `Y` se crea una sala de espera `Y` otro usuario se unirá a esta sala `Y` el creador de la sala le da a "Comenzar Partida" `Entonces`, el juego comenzará y el tablero se generará|Completado|
||AC-5.2 Crear una partida contra la máquina exitoso|`Dado` un usuario que ha iniciado sesión `Cuando` doy click en "Crear Partida" -> "Contra la máquina" `Y` selecciono la dificultad de la máquina `Y` le doy a "Comenzar Partida" `Entonces`. el juego comenzará y el tablero se generará|En progreso|
|HU-6 Movimientos del jugador negras|AC-6.1 Movimiento jugador negras exitoso |`Dado` un juego en curso y no hay un ganador o empate `Cuando` sea el turno del jugador negras `Y` se realize un movimiento válido `Entonces` la pieza debe moverse a la casilla seleccionada `Y` el turno cambia a ser del jugador blancas|Completado|
| |AC-6.2  Movimiento jugador negras fallido  |`Dado` un juego en curso `Y` no hay un ganador o empate `Cuando` sea el turno del jugador negras `Y` se realice un movimiento inválido `Y` luego hago un movimiento ilegal `Entonces` la pieza no debe moverse `Y` el turno del jugador no cambia |Completado|
||AC-6.3 Capturar una pieza exitoso del jugador negras |`Dado` un juego en curso `Y` no hay un ganador o empate `Cuando` sea el turno del jugador negras `Y` se seleccione una casilla del jugador blancas `Y` sea un movimiento válido  `Entonces` la pieza del jugador negras se moverá detrás de la pieza capturada `Y` la pieza del jugador blancas se remueve del tablero `Y` el turno cambia a jugador blancas. |Completado|
||AC-6.4 Capturar múltiples piezas exitoso del jugador negras |`Dado` un juego en curso `Y` no hay un ganador o empate `Cuando` sea el turno del jugador negras `Y` se seleccione una casilla del jugador blancas `Y` sea un movimiento válido `Y` pueda seguir capturando más fichas del jugador blancas `Entonces` el sistema deberá permitirme elegir varias casillas para capturar `Y` no cambiar el turno hasta finalizar|Completado|
||AC-6.5 Capturar una pieza fallido del jugador negras |`Dado` un juego en curso `Y` no hay un ganador o empate `Cuando` sea el turno del jugador negras `Y` se seleccione una casilla del jugador blancas `Y` sea un movimiento inválido `Entonces` la pieza no debe moverse `Y` el turno del jugador no cambia |Completado|
